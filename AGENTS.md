## General Rules

* 所有回复、评论、Review、计划、分析默认使用简体中文。
* Always respond in Simplified Chinese.
* Never use English unless explicitly requested.

## Pull Request Review

Review output format:

### 问题描述

说明发现的问题。

### 风险等级

* 阻塞
* 高风险
* 中风险
* 低风险

### 修复建议

给出明确修复方案。

### 示例代码

如有必要给出修复代码。

## Review Scope

重点关注：

* 运行时错误
* 空值异常
* 类型错误
* 安全漏洞
* 性能问题
* 逻辑缺陷
* React/Vue 生命周期问题
* API异常处理

忽略：

* 纯代码风格
* 格式化问题
* 命名偏好
* 无实际价值的重构建议

## Communication

* 回复使用中文。
* Commit Message 使用中文。
* PR Summary 使用中文。
* Code Review 使用中文。
* Issue 分析使用中文。
* Plan 模式输出使用中文。
